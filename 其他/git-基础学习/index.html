Git--基础学习(版本分支)

Apr 25, 2020 · 2 min read

Git的简单的基础命令

  • 配置信息

    • git --version
    • git config --global user.name // 查看用户名
    • git config --global user.email // 查看邮箱
    • 在上面两个命令的结尾加上变量即可配置

  • 与Linux类似的部分

    • cat
    • rm
    • ls
    • echo "11111" > file 或 >> file
    • cd
    • mkdir

  • Git常用命令

    • git init // 初始化一个仓库
    • git status // 查看当前工作树的状态，可以查看分支合并情况，文件修改情况等
    • git add file1 file2 // 添加到暂存区
    • git commit -m "remarks" // 合并到本地版本
    • git remote add origin "repository" // 关联本地库到远程库
      • origin // 指的是远程仓库
    • git pull // 更新本地分支与远程同步
    • git push <远程主机名> <本地分支名> <远程分支名>
      • git push origin master // 远程分支被省略将会把本地的master分支推送到与之相关联的远程分支 如果没有则被新建
      • git push origin ：<远程分支名> // :代表省略，将空的本地分支推送到远程分支，即删除某个远程分支
      • 使用refs/for/<远程分支名> 说明提交之后还需要 code review 才能进行 merge
      • 在本地分支有追踪关系的远程分支(在本地分支只对应一个远程分支时)，可以省略用 git push (origin)
      • git push -u origin master -f // -f 强制推送 -u 默认主机

Git一些概念理解 (参考: git使用教程)

  • 版本库(.git)

    • 如图所示，每次输入git commit -m "备注信息" 提交即一个版本
    • 每个分支都有一个当前的版本
    • 各个版本之间的关系可以一个双向链表()，HEAD 指针指向当前的分支
    • 使用 git log 可以查看所有的分支。当前分支的最后会有一个 (HEAD -> 分支名) 的提示

  • 分支

    • 我理解的就像上面所说的版本之间象一个链表，分支代表链表的一条分支
    • 刚创建时会自动生成master分支（主分支）
    • 多个版本之间能合并，合并即从分支合并到master分支（主分支），合并成功后将分支删除。
    • 合并要先切换到master分支，然后输入 git merge (--no-ff -m "备注" )分支名
    • 合并产生冲突就修改，先用 git status 查看出错位置



  • 进阶命令

    • git diff / git diff HEAD -- file1 // 前者比较所有文件与当前分支的区别 或者比较特定文件的区别
    • git reset HEAD - file1 // 将进入file1从暂存区移除
    • git branch (-r) // 查看当前分支对应的远程分支， r 应该代表的是 remote
    • git branch -d 分支名 // 删除分支
    • git log // 查看HEAD(最新版本及之前的)所有版本 如果从版本10回退到版本9之后 log 显示的只有版本1-9，版本10则不在其中，要想还原到版本10，需要用到 git reset --hard 版本号 ,而版本号通过 git reflog 可以查到
      • git log --pretty=oneline // 显示在每行显示一个版本
      • git log --graph --pretty-oneline 可以查看分支的合并情况
    • git reset --hard HEAD~n // 回退当前版本的前n(n>=1)个版本 HEAD代表当前最新版本
    • git reset --hard 版本号 // 回退到某个版本
    • git reflog // 查询操作记录 可以依靠这个来看不同版本的版本号
    • git checkout
      • git checkout 分支名 // 切换到该分支
      • git checkout -- file1 // 放弃file1在文件区的改动
    • git checkout -b 分支名 // 创建一个新的分支
    • git merge 从分支名 // 需要在主分支下进行，输入该命令合并到主分支
      • git merge (--no-ff -m "备注") 从分支名 // 合并时禁用快速合并(Fast Forward)，会合并过程中会产生一个新的commit，需要备注
    • git stash // 保存现在的工作现场
      • git stash list // 输出所有保存的现场
      • git stash pop // 恢复所保存的现场

常见错误

  • 提交时出现 LF will be replaced by CRLF in

    windows 下的换行符是 CRLF，而 linux 下的 LF
    解决方法：git config --global core.autocrlf false

  • git log 中文备注乱码 参考2

    git config --global core.quotepath false
    git config --global gui.encoding utf-8
    git config --global i18n.commit.encoding utf-8
    git config --global i18n.logoutputencoding utf-8

    • 添加新的环境变量 LESSCHARSET=utf-8
    • 修改之后乱码消失
