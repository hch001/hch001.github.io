GCC

预处理 (preprocessing)

  • -E
  • 替换#define #pragma #include #if #else #elif #ifndef #ifdef 等
  • 替换头文件
  • 生成 .i (for C) .ii (for c++)

一段简单的预处理的例子

发现 test.i 文件中用头文件的内容替换 #include <stdio.h> 和 #include <skiplist/skiplist.h>，并包括一些使用了哪个文件的注释（可以通过预处理命令检查使用的文件是否正确）

头文件查找的顺序如下s:

  • 当前目录 (使用 “” 优先查找)
  • -I<Directory> 指定的目录
  • /usr/local/include
  • /usr/include

编译 (compile)

  • -S
  • 将预处理后的源代码编译成汇编代码
  • 生成 .s 文件

汇编 (Assemble)

  • -c
  • 将汇编代码转化为机器码（不可执行）
  • 生成 .o 二进制的目标文件或者 .so 动态库( -shared 使用的是相对路径 )或者 .a 静态库

链接(Link)

  • 将一个或多个目标文件连接起来得到一个可执行文件
  • 这一步指定使用的库文件的路径 -L
  • 指定静态库 .a ，使用 -Iskiplist 来加载库文件目录下的(libskiplist.a)文件
  • 指定动态库 .so
    • 使用动态库时的查找路径，/lib /usr/lib ( /usr/local/lib 不属于默认路径 )64位系统还要加上/lib64 /usr/lib64等，可以修改/etc/ld.so.conf来配置路径，需要用ldconfig生效
    • 可以通过 ldd 来查询使用的动态库
    • 运行时才链接，size 比较小
  • 可以使用全称来加载 gcc test.c libskiplist.a -o test

技巧

const 关键字

  • Read-only

    const在C语言中是只读的，不可直接修改值，如果使用指针强行修改，那么结果不确定。gcc里的值会被修改，而clang不会。

静态 全局 extern

  • static 指的是可见性（附带静态存储的效果）
    • static 修饰函数内的局部变量，数据存储在静态区直到程序结束（生存周期为静态），但仅在函数内可访问（可见性）
    • static 修饰函数或者全局变量，数据存储在静态区直到程序结束（生存周期为静态），但仅在本文件内可访问（可见性）
  • 全局指的是全局可见
    • 存储在静态区
    • 全局可见
  • extern 说明在其他文件中查找，与全局无关
    • 说明一个变量在其他文件中定义过，使用extern即在其他文件模块中查找

文件缓冲区大小与物理块

读取文件内容常常需要缓冲区（用户区）来存储，大小一般设置为物理块的大小。原因是物理块是系统分配的基本单位，每个文件占有多个块，每个块只能属于一个文件（不存在一个物理块的一部分属于文件A另一部分属于文件B），若一个物理块大小为1K，文件A有2050字节，那么文件A占有3个物理块，实际占有3K的大小。

若一个文件为3000 Bytes，物理块大小为1KB，设置buff1大小为1024，buff2大小为1500。若使用buff1，那么一共读取三次，每次跨一个物理块，共跨三个物理块；若使用buff2，读取两次，每次跨两个块，共跨4个物理块。

所以缓冲区设置为物理块大小，与字节对齐类似的思想。

使用指针替换数组的陷阱

来源《C专家编程( Expert C Programming)》

上面main.c要使用a.c中的p，例如 p[1]，将不会得到预期的输出，甚至可能segmentation fault。错误流程如下：先找到p指针在栈中的地址addr0，然后根据addr0取出p的内容addr1，addr1加上偏移量得到最终目的数据的地址，最后根据该地址取出数据。正确流程：使用p数组的首地址addr0加上偏移量得到目的数据的地址，取出。区别在于使用指针模拟数据在这种情况下会取出数组的一个指针长度的数据作为addr1，并根据addr1加上偏移量得到目的数据。
