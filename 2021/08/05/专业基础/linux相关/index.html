区别几个 bin 路径

/sbin /bin /usr/bin /usr/sbin /usr/local/bin /usr/local/sbin 的区别：

  • 在根目录下的 /sbin 和 /bin 存放系统必需的命令
  • s代表的是 system，sbin 和 bin 的区别在于是不是系统管理员所需要的命令，前者虽然是系统管理员的命令但不代表普通用户无法使用，如: ping和ifconfig
  • 根目录 / 与 /usr/ 下的区别在于，前者是必备的而后者不是，一般存放自行添加的软件
  • /usr/ 与 /usr/local/ 下的区别在于，前者是包含系统预装的软件，后者一般是用户自己装的，前者可能根据系统的更新而更新

软连接

硬链接复制了一样的 inode (内容和原文件完全一样) ，软链接创建的只是包含了原文件的 inode 号。在修改两种链接文件的内容时，执行的操作最后都会写入到磁盘的同一块区域。

  • 删除原文件后，硬链接仍存有完全一致的 inode，链接计数仍不为0，文件不会被删除，但软链接存储的 inode 就变成了无效的 inode。
  • 针对特殊的目录文件（相对于普通文件多了文件列表），其中包含 . 和 .. 的 inode号，所以如果是硬链接复制一份信息的话，在链接的目录下就无法返回上一级 ..，因为不知道是要返回链接文件本身的上一级目录还是原文件的inode信息中的上一级目录，所以目录链接只能是软链接。
