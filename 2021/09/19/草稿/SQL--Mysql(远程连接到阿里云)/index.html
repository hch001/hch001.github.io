MySQL部署问题

Created2021-09-19|Updated2021-09-18|MySQL
|Word count:284|Reading time:1min|Post View:

在阿里云上配置mysql遇到的错误

  • 修改阿里云的防火墙机制

    添加自定义TCP 端口3306(默认)即可

  • mysql默认绑定了3306端口只能通过localhost访问

    cd /etc/mysql/mysql.conf.d
    vi mysqld.conf
    注释掉里面的 bind-address=127.0.0.1

  • 改变mysql库user表中的某个用户(root)允许的主机

  • Client does not support authentication protocol requested by server 参考

sock出错

在linux下部署mysql，出现类似 cannot connect to …, permission denied.的问题，应该是sock文件或者所在文件夹用户没有权限访问，或者不存在该文件，若不存在，查找mysql的配置文件找到sock文件的路径做一个链接即可，若是权限问题使用chmod该权限即可。
