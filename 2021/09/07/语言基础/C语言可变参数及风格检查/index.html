C语言可变参数引起的思考

Created2021-09-07|Updated2021-09-09|语言基础
|Word count:991|Reading time:3min|Post View:

简介

偶然查找printf这个C语言经典方法的时候发现的问题。从printf想到了C语言如何支持可变参数，想到再由可变参数想到函数的形式参数的压栈顺序，又了解到32位系统和64位系统的表现出来的压栈顺序不同（只是输出地址的话看上去不同）。

可变参数

C语言的函数是支持可变参数的。思想如下：调用函数的时候需要执行返回地址压栈、参数压栈，这时候获取参数实际上就是需要一个参数的起始地址和长度(或者说是类型)，有了这两个东西就能得到一个参数的值。

不同系统之间有差别

系统 情况 顺序
windows 所有情况 压栈从右到左
mac os 函数不含可变参数 压栈从左到右
mac os 函数含可变参数 从左到右，有偏移量，形参之间不连续
linux 32位 函数不含可变参数 压栈从左到右
linux 32位 函数含有可变参数 压栈从右到左
linux 64位 函数不含可变参数 压栈从左到右
linux 64位 函数含有可变参数 从右到左，但是有填充

压栈顺序和支持可变参数的必然联系

从右往左压栈可以通过第一个固定参数来找到其余可变参数（通过偏移，指针+一段偏移）
