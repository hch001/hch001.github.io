模式 描述
^ 匹配输入字符串的开始位置。如果设置了 RegExp 对象的 Multiline 属性，^ 也匹配 ‘\n’ 或 ‘\r’ 之后的位置。
$ 匹配输入字符串的结束位置。如果设置了RegExp 对象的 Multiline 属性，$ 也匹配 ‘\n’ 或 ‘\r’ 之前的位置。
. 匹配除 “\n” 之外的任何单个字符。要匹配包括 ‘\n’ 在内的任何字符，请使用像 ‘[.\n]’ 的模式。
[…] 字符集合。匹配所包含的任意一个字符。例如， ‘[abc]’ 可以匹配 “plain” 中的 ‘a’。
[^…] 负值字符集合。匹配未包含的任意字符。例如， ‘[^abc]’ 可以匹配 “plain” 中的’p’。
p1|p2|p3 匹配 p1 或 p2 或 p3。例如，’z
* 匹配前面的子表达式零次或多次。例如，zo* 能匹配 “z” 以及 “zoo”。* 等价于{0,}。
+ 匹配前面的子表达式一次或多次。例如，’zo+’ 能匹配 “zo” 以及 “zoo”，但不能匹配 “z”。+ 等价于 {1,}。
{n} n 是一个非负整数。匹配确定的 n 次。例如，’o{2}’ 不能匹配 “Bob” 中的 ‘o’，但是能匹配 “food” 中的两个 o。
{n,m} m 和 n 均为非负整数，其中n <= m。最少匹配 n 次且最多匹配 m 次。

使用全文本搜索

  • mysql 支持两种数据库引擎 MyISAM 和 Innodb，只有前者支持全文本搜索
  • 应该在导入数据完毕后再定义为 fulltext，提高效率
  • 使用 Match() 和 Against() 进行搜索
  • 与使用正则表达式的区别在于这个返回的结果是进行排序过的，词出现的频率，出现的位置等

一些注意的点

  • mysql 匹配时不区分大小写

  • 判断空值用 is null

  • 表名和保留字相同时需要用反引号

  • ‘’ 空串和 null 是不同的，not null 可以指定空串

  • 在命令行中使用 delimiter 新的分隔符 来指定新的分隔符（针对出现多个”;”的情况）

  • 使用 auto_increment 时，需要插入数据则该字段可以设置为 null(自动添加) 或者 自己指定值。自动添加的值为上一次插入的值加1，上一次如果是自己指定的值也一样

  • = 与 := 前者是判等后者是赋值

  • 使用 jdbc 时去除警告的方法

  • MYSQL表数据导到文件时出现The MySQL server is running with the –secure-file-priv option
